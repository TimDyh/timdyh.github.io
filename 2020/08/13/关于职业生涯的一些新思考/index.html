关于职业生涯的一些新思考

发表于 2020-08-13 分类于 随笔 阅读次数：
本文字数： 1.4k 阅读时长 ≈ 1 分钟

我在之前写的《职业规划设计》中提到，当时的自己对于到底是读研还是工作还不能给出一个坚定的答复，希望能通过在阿里的这段实习经历找到属于自己的答案。今天，我刚刚结束了转正答辩，整体还算顺利，也基本确定了毕业后直接工作的计划。或许这也一直是我内心的声音。

其实今天是信息量和思考量都很大的一天，除了答辩之外，还有两件事值得一提。

一是听了组内一位师兄关于局部二级索引的技术分享。说实话我全程没有听懂多少，因为之前也没有了解过存储相关的底层原理。虽然不懂技术细节，但并不妨碍我去听其他人之间的讨论。可能是由于大家水平基本都在一个 level 上吧，感觉整个交流氛围热烈而舒适。大家都怀着诚恳的心态，或是提问，或是回答。看着这么多技术大牛你来我往地互动着，而自己却不清楚他们具体说的是什么，还是有那么一丢丢的失落和自卑。这些个 P7、P8、P9 们还真不是盖的，能明显感觉到他们真枪实弹积累的技术底蕴，以及自己和他们的差距。

二是晚上和组里的几个老哥一起撸串聊的一些东西，印象最深的，是一个最近刚入职的老哥。虽然互联网行业跳槽几乎是家常便饭了，但他并不是那种为了跳槽而跳槽，或者只是为了加薪升职。他说他在上一家公司已经没有什么可以再做的了，而且那边感觉比较休闲，于是决定来这边做点更有挑战的事，同时也能多体验一下不同的环境。我当时就很佩服他这种不安于现状、敢于拥抱变化、对自己有要求的态度。然后我又了解到他原来并不是做存储的，于是我问他来这里感觉工作好上手吗？因为我也是对存储几乎一无所知，所以想问问一个有工作经历的人的感受。他说其实给你规划好了事情让你去做是没有什么难度的，真正难的是你要在工作中有自己的思考和总结，自主地寻找当前有哪些问题需要去解决，并给出一个方案去实现。我不能再认同这个观点。能来到阿里的人，我相信在专业技术上都是过硬的。别人给你规划了一套方案，基本上没有做不出来的问题。就比如我实习期间做的项目，就是师兄师姐早早就考虑过的事，只不过现在具体实现由我来完成而已。虽然这不是一个那么简单的项目，但有他们指导我要做哪些事，也就能按部就班地去执行了。所以他这么一说，我感觉我原来思考事情的格局就有点小了。其实技术上的问题都不算是要担心的问题，因为这是可以一点点学习积累的，真正要思考的是如何主观能动地去发现问题、解决问题，有主人翁意识。我想这就是所谓的：

If not me, who? If not now, when?

但话说回来，这些上层的思考还是需要由下层的技术水平作为支撑的。想得再牛逼，做不出来还是空谈。况且我认为一个人如果技术上不行，也不太可能会有多么深入的思考，因为他对很多东西根本就没有概念。

然后我又想到今天答辩的时候，面试官给我提的建议。他说我对于底层的技术原理和系统整体的把握还欠缺一些，这点我是非常服气的，果然一眼就被看穿了。确实这两个月由于时间比较紧，我基本都聚焦在负载均衡这块，对于其他方面没有了解多少。而且我的师兄在我入职的第一天就和我说过，虽然你目前做的是这个项目，但不要把自己局限在这块，整个系统都要了解一些。现在想想，没有一句假话。

所以我觉得我在这段实习结束之后，需要花一整块的时间静下心来充充电，恶补一下数据库、分布式存储相关的东西，在正式入职之前尽可能地提高一波知识水平。而长远来看，也希望自己能像转正答辩中提到的那样，有朝一日成长为分布式存储领域的专家，为团队、公司和社会贡献力量。
